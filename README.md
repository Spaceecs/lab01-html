# Практична робота №9-10: Асинхронна робота з даними у JavaScript

**Тема:** Робота з `fetch()`, JSON, рендеринг каталогу та керування станами інтерфейсу.  
**Виконав:** Боцяновський Олександр

## Реалізований функціонал:

1. **База даних в JSON:** Створено локальну базу даних `data/items.json` з 10 курсів навчання. Кожен об'єкт містить `id`, `title`, `category`, `description`, `price`, `rating`, `level` та посилання на `image`.
2. **Асинхронний Fetch:** Написано стійку конструкцію `async/await` з обробкою помилок за допомогою блоків `try/catch`.
3. **Стани UX:** \* **Loading:** Поки дані завантажуються, працює CSS-анімація кругового лоадера.
   - **Error:** У разі помилки або відсутності файлу виводиться блок з описом проблеми.
   - **Empty Result:** Якщо пошуковий запит або комбінація фільтру не знайшли збігів, виводиться повідомлення про відсутність результатів.
4. **Шаблонізація DOM:** Картки створюються повністю через JavaScript динамічно на основі масиву об'єктів за допомогою шаблонних рядків.
5. **Комбіновані фільтри:** Пошукова система (Live-Search) працює одночасно зі зміною вибору категорії та селектором сортування цін/рейтингу, не перебиваючи стани один одного.
6. **Робота з LocalStorage (Обране):** Натискання на кнопку серця додає або видаляє `id` елемента з локального масиву сховища браузера. Стан зберігається при оновленні сторінки.
7. **Пагінація "Показати ще":** Каталог виводить перші 4 картки, при натисканні на кнопку додаються наступні порції даних без збоїв у фільтрах.
8. **Динамічна модалка деталей:** Клік на кнопку "Деталі" відкриває модальне вікно, яке зчитує дані з конкретного об'єкта та рендерить розширений опис курсу.

## Інструкція запуску:

- Відкрити проект через плагін редактора VS Code **Live Server** (`index.html` або `pages/catalog.html`).
