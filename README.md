**Дисципліна:** Створення Web-сайтів (Front-end) / Web-програмування

## Опис виконаної роботи

У межах практичної роботи створено клієнтську логіку на мові чистий JavaScript (Vanilla JS) для раніше розробленого багатосторінкового мінісайту. Уся робота з DOM-вузлами організована без використання inline-атрибутів через делегування подій у зовнішніх підключених модулях.

### Реалізовані сценарії та механіки:

1. **Статична та гнучка ініціалізація:** Весь код розділено на логічні функції (`ui.js` та `contact.js`), робота яких починається синхронно після події `DOMContentLoaded`. Додано перевірки на існування елементів на сторінці.
2. **Автоматична підсвітка меню:** Завдяки аналізу `window.location.pathname`, активний пункт меню отримує клас `.is-active` на будь-якій сторінці автоматично.
3. **Мобільна адаптивна навігація:** Реалізовано функціональну кнопку-бургер (`.menu-toggle`), яка керує відображенням меню на мобільних пристроях через стан атрибута `aria-expanded`.
4. **Збереження теми інтерфейсу:** Перемикач світлої/темної теми змінює класи на рівні тега `<body>`. Поточний вибір користувача записується у `localStorage` (`siteTheme`), що дозволяє відновлювати дизайн після повного перезавантаження сторінки.
5. **Покращення UX:** Додано кнопку плавного повернення вгору (`.back-to-top`), що з'являється при прокручуванні сторінки нижче за 300px. Розрахунок року у футері виконується об'єктом `Date()`.
6. **Інтерактивні блоки структури:** Реалізовано блок розгортання відповідей FAQ (Акордеон) та систему живого пошуку (фільтрації тексту) по таблиці розкладу занять.
7. **Компонент Lightbox:** Натискання на зображення галереї відкриває адаптивне модальне вікно без деформацій верстки з можливістю закриття за кнопкою або оверлеєм.
8. **Валідація форми та чернетки:**
   - Форма на `contact.html` повністю обробляється через JS (`preventDefault()`).
   - Впроваджено валідацію на стороні клієнта: довжина імені, маска регулярного виразу для пошти, наявність повідомлення та чекбокс. Текст помилки виводиться безпосередньо під невалідними полями.
   - Текстова область має лічильник ліміту символів (`0 / 200`).
   - Будь-які зміни форми автоматично оновлюють чернетку в `localStorage` (`contactDraft`). При успішному надсиланні дані збираються через `FormData`, виводяться на екран у вигляді звіту, а чернетка очищується.

## Інструкція із запуску проекту

Проект працює безпосередньо у браузері користувача.

### Запуск через VS Code (Рекомендовано)

1. Переконайтеся, що в редакторі активовано плагін **Live Server**.
2. Відкрийте кореневу директорію репозиторію.
3. Клацніть правою кнопкою миші на файл `index.html` в папці `lab01-html` та оберіть **"Open with Live Server"**.
